\begin{abstract}
Modern agent frameworks encode tool-using behavior as explicit workflow graphs:
nodes represent LLM calls, tools, routers, or human approval steps; edges
represent possible control-flow transitions. Most ``agent safety'' products
enforce policies at runtime by intercepting calls, adding latency and detecting
violations only after execution is underway. We present \textit{Agentproof}, a
static verification approach that treats an agent workflow as a formal directed
graph and checks safety properties \emph{before deployment}. Agentproof provides
(i) a unified abstract graph model across popular frameworks, (ii) extractors
for several agent orchestration libraries, (iii) structural analyses for common
workflow defects (unreachable nodes, dead ends, missing human gates), and (iv) a
small temporal policy DSL compiled into deterministic finite automata for fast
event-trace evaluation. We demonstrate the approach on representative workflow
graphs built using real framework APIs and report microbenchmarks for extraction
and monitor compilation/evaluation.
\end{abstract}
