# Contributing

## Development Standards
- Keep core transition logic deterministic.
- Add/update docs for behavior or policy changes.
- Include tests for new rule types and failure modes.

## Pull Request Requirements
- Problem statement and design summary.
- Risk analysis for security/verification impact.
- Evidence: test outputs and replay determinism check.

## Code Review Checklist
- Schema compatibility maintained.
- Policy/compiler changes reviewed for semantic correctness.
- Error messages remain structured and actionable.

## Commit Hygiene
- Small, focused commits.
- Clear titles prefixed by area (e.g., `docs:`, `core:`, `verify:`).

## Security Disclosure
Report vulnerabilities privately to project maintainers; avoid public issue disclosure before triage and patch.
